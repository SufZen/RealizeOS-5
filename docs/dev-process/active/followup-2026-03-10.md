These failures all exist in tests written before this session. None are caused by our changes.

#### A. `tests/test_skill_detector.py` — 8 failures

All 8 tests fail. The `detect_skill` function has a YAML loading issue where
the skill files aren't being parsed correctly by the test fixtures.

| Test | Error |
|------|-------|
| `test_detect_skill_v1` | YAML skill not detected |
| `test_detect_skill_v2` | YAML skill not detected |
| `test_no_skill_for_unrelated_message` | YAML skill not detected |
| `test_empty_skills_directory` | YAML skill not detected |
| `test_nonexistent_skills_directory` | YAML skill not detected |
| `test_skill_trigger_case_insensitive` | YAML skill not detected |
| `test_v1_skill_has_pipeline` | YAML skill not detected |
| `test_v2_skill_has_steps` | YAML skill not detected |

**Root cause:** Likely a path or parsing issue in `realize_core/skills/detector.py`.
The test creates temp YAML files but the detector may not be finding them.

**Fix hint:** Check how `detect_skill()` discovers and reads YAML files.

#### B. `tests/test_skill_executor.py` — 4 failures

| Test | Error |
|------|-------|
| `test_v1_single_agent` | `AttributeError: module 'realize_core.skills.executor' does not have the attribute 'call_claude'` |
| `test_v1_multi_agent_pipeline` | Same |
| `test_v1_empty_pipeline` | Same |
| `test_v2_agent_step` | Same |

**Root cause:** The test mocks `realize_core.skills.executor.call_claude` but the executor
module was refactored to use the provider registry instead of importing `call_claude` directly.
The mock target path is stale.

**Fix hint:** Update the mock target to match the current import path, or use
`realize_core.llm.claude_client.call_claude` or similar.

#### C. `tests/test_prompt_builder.py` — 4 failures

| Test | Error |
|------|-------|
| `test_includes_brand` | `AssertionError` — brand content not found in prompt |
| `test_missing_identity_file_still_builds` | `AssertionError` — expected behavior mismatch |
| `test_agent_not_found_no_crash` | `AssertionError` — expected behavior mismatch |
| `test_brand_layer` | `AssertionError` — brand layer content missing |

**Root cause:** The prompt builder's brand file loading changed (directory structure or
file naming) but the tests still expect the old format.

**Fix hint:** Check `realize_core/prompt/builder.py` for how brand files are loaded
and update test fixtures to match.

#### D. `tests/test_scaffold.py` — 2 failures

| Test | Error |
|------|-------|
| `test_creates_readme` | `UnicodeDecodeError: 'charmap' codec can't decode byte 0x90` |
| `test_force_overwrites` | Same |

**Root cause:** Windows cp1252 encoding issue. The scaffold template contains UTF-8
characters (likely emoji) that can't be decoded with the default Windows codepage.

**Fix hint:** Add `encoding="utf-8"` to the `open()` calls in scaffold.py, or set
the test's locale. This is a Windows-specific issue.

#### E. `tests/test_kb_indexer.py` — 1 failure

| Test | Error |
|------|-------|
| `test_discovers_fabric_dirs` | `AssertionError: assert 'F-foundations' in ['systems\\brand1\\F-foundations', ...]` |

**Root cause:** Path separator mismatch. Test expects Unix-style paths (`F-foundations`)
but Windows returns backslash paths (`systems\\brand1\\F-foundations`).

**Fix hint:** Use `Path` objects or `os.path.basename()` in the assertion instead of
raw string comparison.

#### F. `tests/test_llm_router.py` — 2 failures

| Test | Error |
|------|-------|
| `test_content_draft` | `AssertionError: assert 'reasoning' == 'content'` |
| `test_no_signals_returns_none` | `AttributeError: module does not have attribute 'get_feedback_signals'` |

**Root cause:** The classifier boundary for "write a blog post" shifted to `reasoning`
instead of `content`. The `get_feedback_signals` function was removed during refactoring.

**Fix hint:** Update the expected classification in the test, and either restore
`get_feedback_signals` or remove the test that references it.
